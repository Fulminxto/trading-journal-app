# VOLTIS Motion Language

## Purpose

This document defines how VOLTIS moves.

Motion must communicate state, continuity, precision and control.

It must never exist only for decoration.

## Personality

VOLTIS motion is:

- precise
- controlled
- confident
- elegant
- responsive
- solid
- purposeful

VOLTIS motion is never:

- playful
- theatrical
- elastic
- chaotic
- decorative
- distracting
- unnecessarily slow

## Core principle

The user should feel the quality of the motion without being forced to notice it.

## Motion goals

Every motion must improve at least one of these:

- orientation
- hierarchy
- continuity
- feedback
- perceived responsiveness
- understanding of state changes

If it improves none of them, it should not be added.

## Physical character

Elements have weight.

They do not float without reason.

Movement should suggest:

- mass
- friction
- resistance
- controlled acceleration
- controlled deceleration

Avoid exaggerated bounce and overshoot.

## Operational priority

VOLTIS is a trading operating system.

Motion must never slow down:

- data entry
- navigation
- analysis
- trade review
- account management

Function always comes before spectacle.

## Reduced motion

All important interactions must remain understandable and usable when reduced motion is enabled.
